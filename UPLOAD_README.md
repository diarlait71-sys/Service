# Загрузчик данных в Supabase

## 📋 Описание
Скрипт `upload_to_supabase.py` загружает данные из двух источников в Supabase:
- **Бюллетени** (PDF) → таблица `bulletin_rules`
- **Bitrix экспорты** (Excel .xlsx) → таблица `deals`

## ✅ Что сделано

### Файлы созданы:
- ✓ `upload_to_supabase.py` — основной скрипт со всеми функциями парсинга

### Зависимости установлены:
- ✓ `pdfplumber` (0.11.9) — чтение PDF таблиц
- ✓ `openpyxl` (3.1.5) — чтение Excel
- ✓ `pandas` (2.2.3) — работа с табличными данными
- ✓ `requests` (2.33.1) — HTTP запросы к Supabase

## 🚀 Запуск

```bash
python upload_to_supabase.py
```

Или используйте батник (если создадите):

```bash
start_upload.bat
```

## ⚙️ Конфигурация

**В файле `upload_to_supabase.py` отредактируйте:**

1. **SUPABASE_URL** — если отличается (уже установлен правильный URL)

2. **SUPABASE_KEY** — замените на ваш API ключ:
   - Перейдите на https://app.supabase.com
   - Settings → API
   - Скопируйте `anon key` или `service_role key`

   ```python
   SUPABASE_KEY = "ваш_ключ_здесь"
   ```

3. **PDF_FOLDER** — папка с бюллетенями (уже правильно указана)

4. **EXCEL_FOLDER** — папка с Bitrix экспортами (уже правильно указана)

5. **BRAND_KEYWORDS** — добавьте ваши бренды при необходимости

6. **DC_ALIASES** — алиасы дилерских центров

## 📊 Маппинг колонок Excel

Если названия колонок в вашем Excel отличаются, отредактируйте `BITRIX_COL_MAP`:

```python
BITRIX_COL_MAP = {
    "Ваша колонка": "наше поле",
    "VIN":          "vin",
    "Марка":        "brand_raw",
    # и т.д.
}
```

## 🔧 Маппинг колонок PDF

Если структура таблицы в PDF отличается, отредактируйте `COL_MAP` в функции `parse_bulletin_pdf()`:

```python
COL_MAP = {
    "ключевое слово в заголовке": "поле",
    "модел": "model",
    "дц":    "dc_pay",
    # и т.д.
}
```

## ✨ Фишки

- **Автопарсинг дат** — распознаёт форматы: `dd.mm.yyyy`, `yyyy-mm-dd`, `dd/mm/yyyy`
- **Извлечение даты из имени файла** — если имя содержит `_2025_04`, будет использована дата `2025-04-01`
- **Автоопределение бренда** — по ключевым словам в имени файла
- **Дедупликация** — по VIN для сделок
- **Batch обработка** — загружает данные порциями по 100 записей
- **Обработка ошибок** — выводит информативные сообщения об ошибках

## 🐛 Troubleshooting

### Ошибка: "Папка не найдена"
Проверьте пути в `PDF_FOLDER` и `EXCEL_FOLDER`

### Ошибка: "Ошибка 401: Unauthorized"
Проверьте `SUPABASE_KEY` — ключ неправильный или истёк

### Ошибка: "Лист 'report': нет колонки VIN"
В вашем Excel нет колонки с именем, содержащим "vin". Отредактируйте `BITRIX_COL_MAP`

### Правила не загружаются из PDF
Проверьте структуру таблицы в PDF. Может потребоваться отредактировать `COL_MAP` в функции `parse_bulletin_pdf()`
